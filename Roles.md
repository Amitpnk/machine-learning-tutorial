**Core ML & Data roles** are the most common entry points. Data Scientists focus on experimentation and analysis (notebooks, stats, insights), while ML Engineers focus on building and shipping production models. Data Engineers handle the plumbing — pipelines, data lakes, and ingestion. AI Engineers are the newest category: they build applications *on top of* existing LLMs (ChatGPT, Claude, Gemini) using APIs, RAG, and agents without necessarily training models from scratch.

**Operations & Infra roles** sit between ML and DevOps. MLOps Engineers are closest to what the MLA-C01 cert validates — CI/CD, SageMaker Pipelines, model monitoring, and deployment automation. ML Platform Engineers build the internal tooling (feature stores, experiment tracking) that the rest of the ML team uses.

**Specialized roles** go deep in one domain — NLP, Computer Vision, or Reinforcement Learning. Research Scientists are at the frontier, typically requiring a PhD and working on novel algorithms rather than production systems.

**Key distinction worth noting for your course:**
- Data Scientist → *"Can we solve this problem with ML?"*
- ML Engineer → *"How do we build and scale this model?"*
- AI Engineer → *"How do we build a product using existing AI?"*
- MLOps Engineer → *"How do we keep this model running reliably in production?"*
